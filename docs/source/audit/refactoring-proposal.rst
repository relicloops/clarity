Refactoring Proposal Audits
===========================

This is the canonical topic page for refactoring proposals.

Scope
-----

Use this topic for structural cleanup proposals that are justified by current
pain in the codebase, not for speculative rewrites. Tie each proposal to a
specific maintenance cost or scaling problem.

Row schema
----------

- ``Priority``
- ``Audience`` (``user``, ``deployer``, ``maintainer``)
- ``Code target``
- ``Current pain``
- ``Proposed refactor``
- ``Status``
