Documentation Backlog -- 2026-04-14
===================================

This snapshot re-checks the current documentation against the current working
tree and keeps only gaps that are still true.

Method
------

- Compared ``docs/source/`` pages with the live templates, JavaScript, CSS,
  and theme options under ``src/clarity/``.
- Cross-checked ``README.md``, ``docs/Makefile``, and
  ``.github/workflows/ci.yml`` for maintainer-facing gaps.
- Kept file-level references only, to avoid stale line-number citations in a
  moving tree.
- No Sphinx build or browser run was performed in this pass.

Removed After Verification
--------------------------

- The auto-consent example already sets both ``window.__clarityConsent`` and
  ``window.__clarityConsentDecided`` in ``consent.rst``.
- ``chatbot.rst`` already lists horizontal rules in the markdown support
  section.
- The changelog pipeline is no longer a documentation gap: ``changelog.rst``
  now includes the generated ``CHANGELOG.md``.
- The older ``disable chatbot`` finding was stale. The current code exposes
  ``chatbot_enabled`` and gates the chatbot markup in ``layout.html``. The
  remaining issue is missing configuration documentation, which is tracked
  above.
